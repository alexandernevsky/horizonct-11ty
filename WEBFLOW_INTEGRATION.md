# Инструкция по интеграции верстки из Webflow

Этот документ описывает, как интегрировать верстку из вашего проекта Webflow в этот 11ty проект.

## Структура проекта

Проект уже подготовлен для интеграции Webflow верстки:

```
src/
  static/
    webflow/
      css/          # Сюда поместите CSS файлы из Webflow
      js/           # Сюда поместите JS файлы из Webflow
      images/       # Сюда поместите изображения из Webflow
```

## Шаги по интеграции

### 1. Экспорт из Webflow

1. Откройте ваш проект в Webflow
2. Перейдите в **Project Settings** → **Export Code**
3. Скачайте HTML, CSS и JS файлы

### 2. Копирование CSS файлов

1. Найдите все CSS файлы в экспортированном проекте Webflow
2. Скопируйте их в `src/static/webflow/css/`
3. Основной CSS файл обычно называется `normalize.css` или `webflow.css`

### 3. Копирование JS файлов

1. Найдите все JS файлы в экспортированном проекте Webflow
2. Скопируйте их в `src/static/webflow/js/`
3. Основной JS файл обычно называется `webflow.js`

### 4. Копирование изображений

1. Найдите папку с изображениями в экспортированном проекте
2. Скопируйте все изображения в `src/static/webflow/images/`

### 5. Подключение файлов в шаблонах

После копирования файлов, вам нужно обновить шаблоны для их подключения.

#### Вариант 1: Автоматическое подключение через front matter

В начале каждой страницы добавьте:

```yaml
---
webflow_css:
  - normalize.css
  - webflow.css
webflow_js:
  - webflow.js
---
```

#### Вариант 2: Ручное подключение в `default.html`

Откройте `src/_includes/default.html` и добавьте ссылки на ваши файлы:

```html
<!-- После строки с Tailwind CSS -->
<link rel="stylesheet" type="text/css" href="/static/webflow/css/normalize.css"/>
<link rel="stylesheet" type="text/css" href="/static/webflow/css/webflow.css"/>

<!-- Перед закрывающим тегом </body> -->
<script src="/static/webflow/js/webflow.js"></script>
```

### 6. Замена HTML разметки

1. Откройте экспортированный HTML файл из Webflow
2. Скопируйте HTML разметку из нужных секций
3. Вставьте её в соответствующие файлы в `src/`:
   - `src/index.html` - главная страница
   - `src/about/index.html` - страница "О нас"
   - `src/careers/index.html` - страница "Карьера"
   - `src/news/index.html` - страница "Новости"
   - `src/customers/index.html` - страница "Клиенты"
   - `src/contact/index.html` - страница "Контакты"

### 7. Адаптация под 11ty

При замене HTML разметки:

- **Статические тексты** можно оставить как есть
- **Динамический контент** замените на Nunjucks переменные:
  - `{{ settings.company_name }}` - название компании
  - `{{ settings.email }}` - email
  - `{{ settings.phone }}` - телефон
  - `{{ navigation.items }}` - элементы навигации
  - `{{ content }}` - контент страницы

- **Изображения** обновите пути:
  - Из: `/images/logo.png`
  - В: `/static/webflow/images/logo.png`

- **Ссылки** проверьте и обновите:
  - Внутренние ссылки: `/about`, `/contact` и т.д.
  - Внешние ссылки оставьте как есть

### 8. Навигация

Навигация уже настроена в `src/_data/navigation.yaml`. Если нужно изменить структуру меню, отредактируйте этот файл.

### 9. Тестирование

1. Запустите проект: `npm start`
2. Откройте `http://localhost:8080`
3. Проверьте все страницы на корректное отображение
4. Проверьте адаптивность на разных устройствах

## Важные замечания

- **Tailwind CSS** уже подключен в проекте. Если вы используете только Webflow стили, можете отключить Tailwind в `default.html`
- **Alpine.js** уже подключен для интерактивности. Webflow JS может конфликтовать с ним - проверьте совместимость
- **Netlify CMS** настроен для управления контентом. После интеграции верстки проверьте, что все поля в CMS работают корректно

## Структура страниц

Все страницы используют общий шаблон `default.html`, который включает:
- Навигацию (`navbar.html`)
- Контент страницы
- Футер (`footer.html`)

Если вам нужны разные шаблоны для разных страниц, создайте их в `src/_includes/` и укажите в front matter:

```yaml
---
layout: custom-layout
---
```

## Поддержка

Если возникнут проблемы при интеграции:
1. Проверьте консоль браузера на ошибки
2. Убедитесь, что все пути к файлам корректны
3. Проверьте, что все файлы скопированы в правильные папки
